## Типы уязвимостей

### Критические (Critical)

- Компрометация ключей шифрования
- Обход аутентификации (GCM, HMAC)
- Утечка секретных данных
- RCE (Remote Code Execution)

**Время исправления:** В течение 7 дней

### Высокие (High)

- Частичная утечка информации
- DoS (Denial of Service)
- Неправильная валидация входных данных

**Время исправления:** В течение 30 дней

### Средние (Medium)

- Проблемы с производительностью
- Недостаточная валидация
- Информационные утечки (не секреты)

**Время исправления:** В течение 90 дней

### Низкие (Low)

- Проблемы с документацией
- Улучшения безопасности
- Best practices

**Время исправления:** В следующем релизе

## Известные ограничения безопасности

### ECB Mode

**⚠️ ВАЖНО:** Режим ECB небезопасен для шифрования нескольких блоков одинаковых данных. Он реализован только для совместимости и тестирования. Для реального использования используйте CBC, CTR или GCM.

### Пароли

- Используйте сильные пароли (минимум 12 символов, смешанный регистр, цифры, символы)
- Используйте PBKDF2 с достаточным количеством итераций (рекомендуется 100,000+)
- Храните ключи в безопасном месте (не в коде, не в Git)

### Ключи

- Никогда не используйте слабые ключи (все нули, последовательные байты)
- Генерируйте ключи через `generate_random_bytes()` или `cryptocore derive`
- Не передавайте ключи через незащищенные каналы

### IV/Nonce

- Никогда не переиспользуйте IV/nonce с одним и тем же ключом
- Используйте криптографически стойкий генератор случайных чисел
- Для GCM используйте 12-байтовый nonce

## Best Practices

### Для разработчиков

1. **Не логируйте секреты:**
   ```python
   # ❌ ПЛОХО
   print(f"Key: {key.hex()}")
   
   # ✅ ХОРОШО
   print("[INFO] Key generated successfully")
   ```

2. **Очищайте чувствительные данные:**
   ```python
   # После использования
   key = b'\x00' * len(key)  # Перезаписать нулями
   ```

3. **Используйте constant-time операции где необходимо:**
   ```python
   # Для сравнения MAC/tag
   import hmac
   hmac.compare_digest(computed_tag, expected_tag)
   ```

4. **Валидируйте входные данные:**
   ```python
   if len(key) != KEY_SIZE:
       raise ValueError(f"Key must be {KEY_SIZE} bytes")
   ```

### Для пользователей

1. **Храните ключи безопасно:**
   - Используйте менеджер паролей
   - Не храните ключи в открытом виде
   - Используйте разные ключи для разных целей

2. **Используйте правильные режимы:**
   - Для конфиденциальности: CBC, CTR
   - Для конфиденциальности + аутентификации: GCM
   - Избегайте ECB для реальных данных

3. **Проверяйте целостность:**
   - Используйте HMAC или GCM для проверки подлинности
   - Проверяйте хеши файлов перед использованием

4. **Обновляйтесь:**
   - Следите за обновлениями безопасности
   - Обновляйте зависимости регулярно
